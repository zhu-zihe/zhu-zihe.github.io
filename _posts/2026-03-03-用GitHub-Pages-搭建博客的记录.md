一直想有个属于自己的博客，可以写写东西、记录生活。这次终于用 GitHub Pages 和 Jekyll 搭起来了，整个过程比想象中简单，记录一下。

## 为什么选 GitHub Pages？

最主要的原因是 **免费**。GitHub Pages 可以直接绑定仓库，支持自定义域名，而且不需要自己买服务器。加上我平时也用 GitHub 存代码，顺便就把博客放这儿了。

另一个原因是 **省心**。不用配置数据库，不用操心安全更新，写完 Markdown 文件推上去就行。

## Jekyll 是什么？

Jekyll 是一个静态网站生成器，可以把 Markdown 文件转换成 HTML 页面。GitHub Pages 原生支持 Jekyll，所以只要按它的目录结构放文件，推送后自动就能生成网站。

我用的是默认的 Minima 主题，简单干净，适合个人博客。

## 搭建步骤（非常简略版）

1. 在 GitHub 上建一个仓库，名字叫 `用户名.github.io`
2. 克隆到本地，创建 `_posts` 文件夹
3. 在里面放 Markdown 文件，命名格式 `年-月-日-标题.md`
4. 推送上去，等几分钟就能访问了

就这么几步，一个博客就出来了。

## 遇到的小问题

最开始的文章格式不对，日期写错了，文章没显示出来。后来查了一下才知道 Jekyll 对文件名和文件头（Front Matter）要求很严格。

还有就是主题的修改。默认主题挺好看，但想加点自己的东西，比如广告代码，需要找到 `_includes/head.html` 文件来添加。一开始不知道，后来查资料才明白 Jekyll 的文件覆盖机制。
